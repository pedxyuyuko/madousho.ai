# src/madousho/ Package

## OVERVIEW

Main package containing CLI entry point, command modules, database models, configuration, and logging infrastructure.

## STRUCTURE

```
src/madousho/
├── __init__.py          # Package exports (get_logger, __version__)
├── _version.py          # Auto-generated by setuptools_scm (DO NOT MODIFY)
├── cli.py               # Typer entry point, flat CLI registration
├── commands/            # CLI commands (serve, verify)
├── config/              # YAML config loader + pydantic models
├── database/            # SQLite connection + base model (WAL mode)
├── logging/             # Loguru setup, get_logger() export
└── models/              # SQLAlchemy ORM models (flow, task)
```

## WHERE TO LOOK

| Component | File | Purpose |
|-----------|------|---------|
| CLI registration | `cli.py` | Top-level command registration via `app.add_typer()` |
| Serve command | `commands/serve.py` | Main server entry point |
| Verify command | `commands/verify.py` | Configuration/model verification |
| Config loader | `config/loader.py` | YAML loading with pydantic-settings |
| Config models | `config/models.py` | Pydantic models for config validation |
| DB connection | `database/connection.py` | SQLite engine with WAL mode |
| Base model | `database/base_model.py` | Common model mixins |
| Flow model | `models/flow.py` | Flow entity definition |
| Task model | `models/task.py` | Task entity definition |
| Logger setup | `logging/config.py` | Loguru configuration |

## CONVENTIONS

- **Flat CLI structure**: All commands registered at top-level via `app.add_typer()` (no nested subcommands)
- **Config via environment**: `MADOUSHO_CONFIG_PATH` set by CLI `--config-path` option
- **Japanese comments**: Present in logging exports and some internal modules
- **All subdirs export via `__init__.py`**: Import key symbols for cleaner imports

## ANTI-PATTERNS

- DO NOT modify `_version.py` (auto-generated from git tags)
- DO NOT create nested CLI commands (flat structure enforced)
- DO NOT import directly from submodules (use package exports)
