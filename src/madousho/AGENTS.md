# src/madousho - Core Package

## OVERVIEW

Main package containing CLI, commands, models, database, config, and logging modules.

## STRUCTURE

```
src/madousho/
├── cli.py              # Typer CLI entry point
├── _version.py         # Auto-generated by setuptools_scm
├── __init__.py         # Exports: __version__, get_logger
├── commands/           # CLI subcommands (serve, verify)
├── config/             # Pydantic models + YAML loader
├── database/           # SQLAlchemy connection singleton
├── logging/            # Loguru configuration
└── models/             # SQLAlchemy ORM models (Flow, Task)
```

## WHERE TO LOOK

| Module | Files | Purpose |
|--------|-------|---------|
| CLI | `cli.py` | Typer app, global options, command registration |
| Commands | `commands/` | `serve` (API server), `verify` (config validation) |
| Config | `config/` | `models.py` (Pydantic), `loader.py` (YAML + caching) |
| Database | `database/` | `connection.py` (singleton), `base_model.py` (Base classes) |
| Models | `models/` | `flow.py`, `task.py` (UUID-based ORM) |
| Logging | `logging/` | `config.py` (Loguru sinks setup) |

## CONVENTIONS

- **Module imports**: Use `from madousho.xxx import yyy` (not `src.madousho`)
- **Singleton access**: `Database.get_instance()`, `get_config()` (lazy init)
- **Context managers**: Use `db.session()` for DB transactions (auto commit/rollback)
- **Logging**: Call `configure_logging()` once at app startup, then use `logger` global
- **Config path**: Set via `MADOUSHO_CONFIG_PATH` env var or `--config-path` CLI option

## ANTI-PATTERNS

- **DO NOT** import from `src.` prefix in application code - use `madousho.` instead
- **DO NOT** create multiple `Database` instances - always use `get_instance()`
- **DO NOT** call `configure_logging()` multiple times - it's one-time setup
- **DO NOT** bypass `db.session()` context manager for DB operations
